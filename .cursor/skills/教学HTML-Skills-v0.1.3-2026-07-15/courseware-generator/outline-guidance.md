# 课件大纲生成指南

本文件只服务本地多页课件生成，不包含外部平台工具协议。

## 目标

把用户需求整理成可直接生成 HTML 的逐页课件大纲。大纲不是教学论文，应优先服务后续页面生成和验收。

## 需要提取的信息

尽量从用户原文提取；缺失但可合理推断时直接给默认值，并在回复中说明。只有缺失信息会影响页数、教学目标或核心互动时，才一次性提问。

```text
课时内容
学段/年级
教材或版本（如用户提供）
教学目标或重难点
课件页数
授课类型
班级学情（可默认中等）
必须包含或禁止的内容
```

## 大纲设计原则

- 每一行对应一页，页码必须连续。
- 页类型至少覆盖封面、讲解/演示、练习或小结；具体按需求调整。
- 每页内容量必须能放进 960×540 画布。
- 交互设计只写静态 HTML/CSS/JS 可实现的方式。
- 禁止设计摄像头、文件上传、多人联网等超出静态前端能力的功能。
- 练习页应说明题型、题量、反馈方式；多题练习默认同页逐题切换。

## artifact-spec.outline 输出格式

```json
"outline": [
  {"id":1,"type":"封面","content":["标题","学科年级","课程主题"],"interaction":"无"},
  {"id":2,"type":"讲解","content":["核心概念","例子"],"interaction":"点击展开重点"}
]
```

`artifact-spec.outline` 需要逐页列出页码、页类型、内容要点、交互设计。每个互动页都要标注自己的 `core-loop`；练习页要说明题型、题量、反馈方式和是否上报分数。封面需要图片时，在 `artifact-spec.assets` 记录 `coverImageSlot`；无外部素材时写明 CSS/SVG/Canvas 自绘。

大纲可用自然语言写图片用途、画面主体与风格倾向，**不得**输出完整生图 prompt，**不得**提前套用 `references/image-generation.md` 的风格库。是否增强 prompt 由 Step 1 真正调用生图工具前按单张图命中判断。

## Step 1 自检

```text
□ outline 已形成并确认，或用户已提供完整逐页结构
□ outline 和 assets 已准备写入 artifact-spec
□ spec 四字段已整理，mode=courseware
□ 每个互动页都有可验收 core-loop
```

## 确认规则

多页课件默认需要用户确认大纲后再生成 HTML。确认点只聚焦页数、页序、内容取舍和关键交互，不要求用户审查实现细节；确认的是内存中的逐页结构，不生成独立文件。

如果用户已经提供完整逐页结构，可直接进入生成，并在最终说明“已按用户给出的逐页结构执行”。
