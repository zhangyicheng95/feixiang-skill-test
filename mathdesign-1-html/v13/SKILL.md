## 一、 渲染规范 (Rendering Specification)
使用mathjax3渲染公式，生成的公式需要满足mathjax3格式，CDN为<script src="https://metis-online.fbcontent.cn/metis-misc/blER0Bn7vsa2JER9IEssf8.js"></script>

**【关键】数学公式渲染规范**：
HTML内容中的数学公式**必须使用 `\(...\)` 作为行内公式定界符**，严禁使用 `$...$` 格式。MathJax默认配置仅识别标准LaTeX定界符，使用错误的定界符会导致公式无法渲染显示。
- ✅ **正确示例**：`若 \(a=b\)`、`\(x+1=6\)`、`\(\pm\sqrt{10}\)`
- ❌ **错误示例**：`若 $a=b$`、`$x+1=6$`、`$\pm\sqrt{10}$`
- 适用范围：HTML中所有数学表达式、变量、符号都必须遵循此规范


## 二、 视觉硬性要求 (Visual Identity)
1. **画布比例参考**：页面设计以 `16:7` 作为视觉排版参考比例，但实际布局必须适应浏览器窗口尺寸。
2. **页面高度控制**：
主页面容器必须使用：height: calc(100vh - 24px);确保在浏览器 UI 存在时仍能完整显示内容。
3. **页面级禁止滚动**：页面根容器必须使用 `overflow: hidden;`，禁止出现浏览器页面滚动条。
4. **内容溢出处理规则**：当内容超过可视区域时，必须通过以下方式解决：
   - Card 内部滚动 (`overflow-y: auto`)
   - Step Navigation（步骤切换）
   - Card Tab 切换
5. **禁止通过页面高度扩展解决内容溢出问题**。
6. **禁止 Emoji**：全篇严禁出现任何表情符号，确保学术严谨感。
7. **禁止图片**：页面中严禁使用 `<img>` 标签或任何背景图片。
8. **透明卡片设计**：
   - 1px 描边：`border: 1px solid ${primaryColor}4D;`（4D 代表 30% 透明度）。
   - 描边颜色必须使用主色调（Primary）。
   - 描边圆角：`border-radius: 16px;`。
   - 禁止填充：`background: transparent;`（严禁背景色块，保持视觉通透）。
9. **字体选择**：几何感无衬线体（Inter, Roboto, PingFang SC）。
10. **内容完整展示**：优先一屏展示所有交互与全部内容，如果溢出需要滑动。

## 四、 布局与代码约束 (防留白逻辑)
1. **比例化布局**：强制使用 `display: grid; grid-template-columns: 2fr 1fr;`。左侧主展示区必须占据 60%-70% 宽度，右侧控制/说明区占据 30%-40% 宽度。
2. **垂直撑满**：右侧卡片区必须使用 `display: flex; flex-direction: column; justify-content: space-between;`，确保卡片内容在垂直方向上均匀分布，消除底部空白。
3. **SVG 动态化**：所有几何图形必须使用 `<svg>` 绘制，并为关键元素添加 CSS 动画（如 `transition` 或 `animate`）。
4. **SVG 填充规则**：UI Card 必须保持无填充，但 SVG 几何图形允许使用 `fill` 颜色进行结构强调或高亮。
5. **全局间距**：设置 `padding: 24px; gap: 24px;` 确保呼吸感。
6. 页面垂直结构必须采用两层布局：
page
 ├ header（标题区域）
 └ main（内容区域）
7.**Header 高度规则**：
header 区域高度应控制在 `80px – 120px` 之间，
禁止 header 占据超过页面高度的 `15%`。
8. **标题区域必须独占一行，左右内容仅在 main 区域内使用 grid 布局。
9. **页面高度安全区**：浏览器可用高度通常小于 `100vh`。页面主容器必须使用 `height: calc(100vh - 24px);` 确保在浏览器 UI 存在时仍能完整显示内容。
10. **整体页面禁止滚动**：页面根容器必须设置 `overflow: hidden;`，禁止页面级滚动条。
11. **Card 内部滚动规则**：当内容超过 Card 高度时必须在 Card 内部滚动而不是扩展页面高度。Card 必须允许 `overflow-y: auto;`。
12. **Card 最大高度**：所有 Card 必须限制为 `max-height: 100%;`，禁止 Card 高度突破主布局容器。
13. **左侧展示区 Card 数量限制**：左侧主展示区同时显示的 Card 数量最多为 `3` 个。超过内容必须通过步骤切换（Step Navigation）或卡片切换展示。
14. **SVG 尺寸限制**：
所有 SVG 必须设置：
max-width: 100%;
max-height: 60%;
object-fit: contain;

SVG 高度必须为 Card 可用高度的 50%–60%，
禁止 SVG 占据 Card 的主要垂直空间。
14. **MathJax 公式尺寸控制**：复杂公式必须使用 display 模式，但视觉高度不得超过两行正文高度，避免公式撑高 Card。
15. **响应式布局规则**：当屏幕宽度小于 `1400px` 时布局改为 `grid-template-columns: 1fr;` 左右区域垂直排列。当屏幕宽度大于 `1400px` 时使用 `grid-template-columns: 2fr 1fr;`。
16. **响应式间距规则**：默认 `padding: 24px; gap: 24px;`。当屏幕高度小于 `900px` 时自动调整为 `padding: 16px; gap: 16px;`。

## 五、 字号系统 (Typography Scale)
1. **一级标题（主标题）**：字号 `42px / 2.6rem`，字重 `700`。用于页面主标题或核心问题标题。
2. **二级标题（模块标题）**：字号 `34px / 2.1rem`，字重 `600`。用于主要卡片标题、章节标题、步骤标题。
3. **三级标题（子模块标题）**：字号 `30px / 1.8rem`，字重 `600`。用于说明区小标题或图形说明标题。
4. **重要信息（关键内容）**：字号不得小于 `28px / 1.6rem`，字重 `500`。适用于交互按钮文字、关键解释、计算步骤、最终结论等核心信息。
5. **正文信息（解释内容）**：字号不得小于 `26px / 1.55rem`，字重 `400`。适用于普通解释文本、推导辅助说明等。
6. **辅助说明（次要信息）**：字号不得小于 `24px / 1.5rem`，字重 `400`。适用于补充说明、提示信息、图形辅助描述。
7. **按钮文字（交互元素）**：字号 `28px / 1.6rem`，字重 `600`。所有交互按钮必须使用该规格，保证可读性与点击清晰度。
8. **数学公式规则（MathJax）**：公式字号必须跟随所在文本层级。标题中的公式使用对应标题字号；正文中的公式使用正文字号；推导步骤中的公式使用 `28px` 关键内容字号。公式视觉尺寸不得小于 `28px`。
9. **行高规则（Line Height）**：所有文本必须设置 `line-height: 1.35 – 1.5`，避免大字号文本造成垂直空间浪费或行间重叠。
10. **字号层级限制**：单个页面同时使用的字号层级最多为 `4` 种，避免视觉层级过多导致排版混乱。
11. **MathJax 默认尺寸基准**：MathJax 公式默认视觉尺寸应与 `28px` 文本接近，并通过 `transform: scale()` 或 MathJax 配置进行微调，避免公式高度明显大于正文。

## 六、 排版间距系统 (Spacing System)

1. **基础单位**
所有排版间距必须基于 `8px` 设计系统进行倍数扩展，常用间距单位为：

8px / 16px / 24px / 32px / 40px

禁止出现随机间距数值。

2. **标题与内容间距**

一级标题 → 内容  
margin-bottom: 32px

二级标题 → 内容  
margin-bottom: 24px

三级标题 → 内容  
margin-bottom: 16px

3. **段落间距**

正文段落之间必须保持：

margin-bottom: 16px – 20px

禁止段落紧贴排版。

4. **公式与文本间距**

行内公式与文字之间保持自然空隙。

独立公式块上下间距：

margin-top: 16px  
margin-bottom: 16px

避免公式与文本挤压。

5. **Card 内部间距**

Card 内容容器必须设置：

padding: 24px

Card 内元素垂直间距建议：

gap: 16px – 20px

6. **SVG 与文字间距**

SVG 图形与说明文字之间必须保持：

margin-top: 16px  
margin-bottom: 16px

避免图形贴边。

7. **按钮区域间距**

按钮组必须与正文保持：

margin-top: 24px

多个按钮之间间距：

gap: 12px – 16px

确保交互区域清晰。
