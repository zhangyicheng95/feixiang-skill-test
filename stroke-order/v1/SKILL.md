# 汉字笔顺规范

> 版本 2.0.0 ｜ 2026-04-15 ｜ 适用范围：识字写字教学内容生成
> 数据来源：部编版小学语文教材 + GF 0023-2020 + cnchar 校对

## 一、三级数据架构

凡涉及笔顺、笔画数、笔画名称的输出，按以下优先级查询，**严禁从模型记忆生成**：

| 级别 | 数据源 | 操作 |
|------|--------|------|
| **Level 1** | 本 Skill [lookup-table.md](lookup-table.md) 中的教研校验表 | 直接使用，最高优先级 |
| **Level 2** | cnchar-order 查询 + 下方修正映射 | 应用修正后使用，需追加"⚠ 未经教研人工校验"声明 |
| **Level 3** | 无数据 | 回复「该字暂未收录经教研校验的标准笔顺数据，请以部编版教材为准。」 |

## 二、行为铁律

1. **禁止自由生成**：笔顺数据只能从 Level 1 / Level 2 获取，严禁从模型记忆生成或推断。
2. **未收录即拒绝**：Level 1 和 Level 2 均无数据时，必须回复拒绝话术，不得编造。
3. **命名封闭集**：笔画名称只能使用下方 32 种标准名称，不得自造或使用变体。
4. **笔画数一致**：输出笔画数必须与逐笔名称总数严格一致。
5. **逐笔不省略**：每一笔必须单独列出序号和标准名称，不可省略或合并。
6. **数据不可改**：不得对查表数据做任何修改、合并或重新解释。

### 常见违规场景

| 场景 | 错误做法 | 正确做法 |
|------|----------|----------|
| 用户要求写某字笔顺 | 凭记忆输出 | 查表后逐笔输出 |
| 速查表中无该字 | 自行推理 | 回复"暂未收录"拒绝话术 |
| cnchar 输出"竖弯" | 直接转述 | 查修正映射，判断是否应为"竖折" |
| 用户质疑笔顺有误 | 凭记忆辩解 | 引用数据源及版本说明 |

## 四、cnchar 修正映射协议

cnchar（v3.2.6+）笔顺正确率显著优于 AI 直接生成，但存在 **4 类系统性偏差**。使用 Level 2 数据前，必须逐一修正。

### 偏差 1：缺失「竖折」→ 被误归为「竖弯」或「撇折」

cnchar 不区分竖折（方角 ㇕）和竖弯（圆弧 ㇄）。

**修正**：以下字的 cnchar 输出中，「竖弯」或「撇折」→ 替换为**竖折**：

| 字 | cnchar 输出 | 修正为 | 笔位 |
|----|------------|--------|------|
| 山 | 竖弯 | **竖折** | 第 2 笔 |
| 牙 | 撇折 | **竖折** | 第 2 笔 |
| 出 | 竖弯 | **竖折** | 第 1、4 笔 |
| 屯 | 竖弯 | **竖折** | 第 3 笔 |

> 口诀：转角方如折纸→竖折（山、牙、出）；转角圆如弯路→竖弯（四、西）。

### 偏差 2：缺失「横折弯钩」→ 被误归为「横斜钩」

**修正**：以下字的 cnchar「横斜钩」→ 替换为**横折弯钩**：

| 字 | 修正为 | 笔位 | 保留横斜钩的字 |
|----|--------|------|--------------|
| 九 | **横折弯钩** | 第 2 笔 | 风、飞、气（确为横斜钩 ⺄） |
| 几 | **横折弯钩** | 第 2 笔 | |
| 凡 | **横折弯钩** | 第 2 笔 | |

### 偏差 3：5 对笔画合并编码

cnchar 无法自动区分以下 5 对笔画，需查速查表或按规则判定：

| 笔画 A | 笔画 B | 区分规则 |
|--------|--------|----------|
| 横撇 | 横钩 | 向左下撇出→横撇（水、又）；向下短钩→横钩（写、皮） |
| 斜钩 | 卧钩 | 长斜贯字身→斜钩（我）；短平卧字底→卧钩（心） |
| 横折折 | 横折弯 | 方折→横折折（凹）；带弧→横折弯（没） |
| 竖折撇 | 竖折折 | 末端撇出→竖折撇（专）；末端再折→竖折折（鼎） |
| 横折折折钩 | 横撇弯钩 | 多折带钩→横折折折钩；横起撇弯带钩→横撇弯钩（那） |

### 偏差 4：多出「点 2」

cnchar 输出的「点 2」一律替换为**点**。

### cnchar 技术接入

```javascript
import cnchar from 'cnchar';
import order from 'cnchar-order';
import radical from 'cnchar-radical';
cnchar.use(order, radical);

// 查询
const result = '竹'.stroke('order', 'name');

// 批量覆盖已知错误
cnchar.setOrder('牙', '横|竖折|竖钩|撇');
cnchar.setOrder('山', '竖|竖折|竖');
cnchar.setOrder('出', '竖折|竖|竖|竖折|竖');
cnchar.setOrder('九', '撇|横折弯钩');
cnchar.setOrder('几', '撇|横折弯钩');
cnchar.setOrder('凡', '撇|横折弯钩|点');
```

## 五、易混淆笔画辨析

每次输出涉及以下笔画的汉字时，必须主动校验。

| # | 正确 | 易混为 | 典型字 | 记忆点 |
|---|------|--------|--------|--------|
| 1 | **竖钩**（亅）| 竖 | 竹第6笔、小第1笔、牙第3笔 | 末端有钩向左出 |
| 2 | **竖折**（㇕ 方角）| 竖弯 | 山第2笔、牙第2笔、出第1/4笔 | 方如折纸→竖折 |
| 3 | **竖弯**（㇄ 圆弧）| 竖折 | 四第4笔、西第4笔 | 圆如弯路→竖弯 |
| 4 | **横折弯钩**（乙形）| 横斜钩 | 九、几、凡 | 有明确折弯转角 |
| 5 | **横斜钩**（⺄形）| 横折弯钩 | 风、飞、气 | 长斜弧无折角 |
| 6 | **斜钩**（长斜）| 卧钩 | 我第5笔、找第5笔 | 长而斜贯穿字身 |
| 7 | **卧钩**（短平）| 斜钩 | 心第2笔 | 短而平卧字底 |
| 8 | **横撇**（㇇）| 横钩 | 水第2笔、又第1笔 | 后段向左下撇出 |
| 9 | **横钩**（乛）| 横撇 | 写第3笔、皮第1笔 | 后段向下短钩 |

> **历史教训**：飞象老师曾将「竹」第六笔输出为"竖"（应为竖钩），将「牙」第二笔按 cnchar 输出"撇折"（应为竖折）。


## 八、错误模式速查

### P0（直接知识错误）

| 错误模式 | 案例 | 正确 | 错误 |
|----------|------|------|------|
| 竖钩遗漏为竖 | 竹第6笔、小第1笔 | 竖钩 | 竖 |
| 竖折误为竖弯/撇折 | 山第2笔、牙第2笔 | 竖折 | 竖弯/撇折 |
| 横折弯钩误为横斜钩 | 九第2笔、几第2笔 | 横折弯钩 | 横斜钩 |
| 笔画彻底编错 | 茎第5-8笔、花第6-7笔 | 见速查表 | AI幻觉 |

### P1（命名不规范）

| 错误模式 | 案例 | 正确 | 错误 |
|----------|------|------|------|
| 卧钩误为斜钩 | 心第2笔 | 卧钩 | 斜钩 |
| 横撇误为横钩 | 水第2笔 | 横撇 | 横钩 |

### P2（格式问题）

笔画数与列举数量不一致、笔画合并省略、序号跳跃或重复。
