# 交互与动效优化策略

本文件处理动画和交互类需求：为静态页面添加动效、升级已有的简单交互、在保护复杂交互逻辑的前提下优化其表现层。

---

## 一、交互诊断

根据前置分析 5.2 的代码特征扫描结果，判断页面的交互现状，决定进入「增强模式」还是「保护模式」。

### 1.1 增强模式

**触发条件**：页面无 JS，或仅有简单 JS（如点击切换 class、显示/隐藏元素）。

**执行方向**：按第三~七节的指导，为页面添加入场动画和适当的交互。

### 1.2 保护模式

**触发条件**：页面包含复杂 JS 交互逻辑，满足以下任一特征：
- 使用 Canvas 或 SVG 做动态渲染
- 存在 `requestAnimationFrame` 或 `setInterval` 驱动的循环
- 存在状态机（多个状态变量 + 状态切换逻辑）
- 存在拖拽/碰撞检测/判定计算等交互算法
- JS 代码量 ≥ 100 行且包含事件驱动的业务逻辑

**执行方向**：严格保护现有交互逻辑，仅优化表现层。参见第二节。

---

## 二、已有交互保护规则

> 仅在「保护模式」下执行。
>
> **此规则优先级高于本文件中的所有其他规则。违反即任务失败。**
>
> **核心原则：原始代码中的逻辑层必须完整保留，能修改的只有排版层。**

### 2.1 逻辑层（必须完整保留）

逻辑层代码直接承载教学目标，修改即破坏教学。纯逻辑代码（状态管理、计算、判定）不占画面空间，更加没有任何改动的理由。

| 类别 | 包含内容 | 为什么不能动 |
|------|----------|-------------|
| 状态管理 | 状态机、状态切换逻辑 | 操作流程就是教学流程 |
| 数据模型 | 题目数组、关卡配置、计分规则、玩家数据结构 | 题目/关卡的数量和内容是教学设计 |
| 计算逻辑 | 频率计算、偏差判定、公式求值、碰撞检测算法 | 数学准确性是底线 |
| 判定规则 | 正确/错误判断、触发条件、阈值比较 | 改了对错就乱了 |
| 反馈机制 | 动画触发条件、音效播放条件、结果展示触发、自动切换逻辑 | 反馈时机是教学设计的一部分 |
| 交互处理 | 拖拽判定、点击响应、滑动处理的**判断和计算部分** | 交互方式就是教学方式 |
| 文字内容 | 所有文本：题目、选项、解析、提示、标签、按钮文字 | 每个字都对应知识点 |
| 数值/公式 | 代码中出现的数字、数学公式、计算常量 | 数学准确性 |
| 图形比例关系 | Canvas/SVG/CSS 绘制的图形的长宽比、相对大小 | 数学图形的比例本身传递信息 |
| 素材引用 | 所有图片 URL、音频 URL、字体 URL | 资源完整性 |
| 动画过程 | CSS/JS 动画的关键帧、缓动函数、时长、步骤序列 | 动态变化本身就是知识点 |

### 2.2 排版层（可以自由调整）

排版层只影响「看起来怎么样」和「放在哪里」，不影响「做什么」和「学到什么」：

| 类别 | 包含内容 | 调整方式 |
|------|----------|----------|
| 容器尺寸 | Canvas 宽高、根容器宽高 | 适配 960×540 |
| 元素定位 | top/left/right/bottom 坐标值 | 适配新画布尺寸 |
| 布局方向 | flex-direction、grid 排列方式 | 纵→横、横→纵重排 |
| 字号/间距 | font-size、padding、margin、gap | 缩放适配 |
| 配色/字体 | color、background、font-family | 统一课件风格 |
| 圆角/阴影/边框 | border-radius、box-shadow、border | 统一视觉细节 |
| 生成范围 | 随机 x/y 的 min/max 值 | 适配 960×540 数值范围 |
| DOM 排列结构 | innerHTML 生成的 HTML 的排列方式和尺寸 | 重排以适配空间（数据内容不可删） |

### 2.3 JS 代码的黄金法则

JS 代码同时包含逻辑层和排版层，**必须精确区分、逐行审查**，禁止整段重写：

- **逻辑层**（`if` 判定、数据操作、状态切换、反馈触发）→ 原样保留
- **排版层**（`window.innerWidth`→`960`、坐标值、字号、布局方向）→ 可以调整
- **innerHTML 中的数据内容**（`${player.name}: ${count}张`）→ 逻辑层，不可删减
- **innerHTML 中的样式属性**（`style="font-size:18px"`）→ 排版层，可以调整

### 2.4 放不下时怎么办

**排版层有无限的调整空间，不存在「放不下所以要删功能」的情况。**

按优先级处理：缩小尺寸 → 重排布局 → 紧凑化 → 浮层化。

**禁止「删除」。弹窗/模态框是覆盖在画面上的浮层，完全不受 960×540 画布限制。**

## 三、增强模式设计原则

> 以下适用于「增强模式」。你拥有充分的创作自由度，本节只提供核心原则和壳框架约束。

### 3.1 核心原则

**内容不变，形式自由。**

用户要求增加交互时，你的任务是：用高互动的方式重新呈现页面已有的教学内容。

- **鼓励大胆革新**：不必拘泥于原有排版和呈现形式，可以彻底重构页面的交互方式——只要教学内容基本一致、能达到教学目的即可
- **自主设计交互**：根据页面内容的特点，自行思考最适合的互动形式（逐步揭示、拖拽排序、点击探索、动画演示、答题闯关……），不限于简单的入场动画
- **教学目的优先**：所有交互都应帮助学生更好地理解内容，而非为动而动

### 3.2 壳框架约束（必须遵守）

以下约束源自壳框架的运行机制，违反会导致功能异常：

- **禁止使用方向键（↑↓←→）和空格键**：壳框架在演示模式下拦截这些按键用于翻页
- **禁止使用 `alert()`/`confirm()`/`prompt()`**：原生弹窗会退出全屏演示模式
- **入场动画不要用 `animation-delay` 做 stagger**：无法配合 restoreState 跳过动画，改用 JS `setTimeout` 控制时序

## 四、壳框架集成要求

凡新增了交互（点击、输入、展开等用户操作）的页面，**必须**实现以下壳框架协议。纯展示页面（只有入场动画）无需实现。

### 4.1 状态保存与恢复（saveState / restoreState）

壳框架翻页时会**直接销毁**当前页面的 iframe，不给页面保存的机会。因此状态必须**即时上报**。

**保存**：每次用户交互导致页面"看起来不同了"时立即调用：

```javascript
window.parent.postMessage({
  type: 'saveState',
  state: { /* 可 JSON.stringify 的纯数据 */ }
}, '*');
```

- 选择了选项 → 立即保存（不能等到点"提交"）
- 输入了文字 → 每次 `input` 事件保存（不能等到点"提交"）
- 展开了内容 / 动画完成 → 保存
- 多次上报无副作用，壳只保留最新一次
- state 只能包含基本类型（字符串、数字、布尔、数组、纯对象）

**恢复**：壳框架往前翻页时，会向页面发送之前保存的状态：

```javascript
window.addEventListener('message', function(e) {
  if (e.data && e.data.type === 'restoreState') {
    // 根据 e.data.state 直接恢复 UI
    // 必须跳过入场动画，直接显示终态
    // 恢复后的页面应与用户离开时完全一致
  }
});
```

### 4.2 拖拽交互的防误翻页

如果新增了拖拽交互，须遵守壳框架的防误翻页协议：

- **拖拽进行时添加 `.dragging` 类**：`mousedown`/`touchstart` 时加，`mouseup`/`touchend` 时移除。壳框架检测此类来阻止翻页
- **拖拽元素和放置区添加 `data-interactive` 属性**：壳框架识别此标记，不在这些元素上触发翻页
- **`<canvas>` 和 `<svg>` 自动识别**：壳框架已将它们标记为交互元素，无需额外处理
- **整页交互**：如果整页都是交互内容，给最外层容器加 `data-interactive`

## 五、点击放映动画的翻页禁用

> 当前置分析 5.2 检测到页面通过 `data-interactive` 禁用了点击翻页时，执行本节。

### 5.1 优化规则

**如果优化后的页面仍然保留了点击触发动画的设计**（无论是保留原始脚本还是重写了动画逻辑），必须同时保留 `data-interactive` 的管理机制：

1. **页面加载时**：在根容器上设置 `data-interactive` 属性，禁用点击翻页
2. **每次点击**：播放下一步动画，不翻页
3. **所有动画步骤播完后**：移除 `data-interactive` 属性，恢复点击翻页
4. **状态恢复时**：如果恢复后动画已全部播完，也要移除 `data-interactive`

参考实现：

```javascript
var r = document.querySelector(".sr");
r.setAttribute("data-interactive", "");

r.addEventListener("click", function () {
  if (c >= S.length) return;
  // 播放第 c 步动画...
  c++;
  window.parent.postMessage({ type: "saveState", state: { animStep: c } }, "*");
  if (c >= S.length) r.removeAttribute("data-interactive");
});

window.addEventListener("message", function (e) {
  if (e.data && e.data.type === "restoreState" && e.data.state) {
    var t = e.data.state.animStep || 0;
    // 跳过动画直接显示终态...
    c = t;
    if (c >= S.length) r.removeAttribute("data-interactive");
  }
});
```

**如果优化后不再使用点击触发动画**（例如改为自动播放或直接展示所有内容），则无需保留 `data-interactive`。

### 5.2 常见错误

| 错误 | 后果 |
|------|------|
| 优化后删掉了 `data-interactive` 管理，但保留了点击动画脚本 | 每次点击既播放动画又翻页，用户只能看到第一步动画就被翻走 |
| 设置了 `data-interactive` 但动画播完后忘记移除 | 动画结束后用户无法通过点击翻到下一页，页面"卡死" |
| `restoreState` 中恢复到已播完状态但未移除 `data-interactive` | 翻回该页时页面"卡死"，无法继续翻页 |
| 根容器选择器与实际 DOM 不匹配（如 `.sr` 改名但脚本没更新） | `setAttribute` 作用在 null 上，翻页禁用失效 |
