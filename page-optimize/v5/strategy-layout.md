# 排版与结构优化策略

本文件处理三类关联需求：**布局修复**（元素错位/重叠）、**排版优化**（间距/层次/布局模式）、**内容充实**（信息量不足）。

---

## 一、布局诊断

根据前置分析 5.2 的代码特征扫描结果，判断页面的布局现状，选择对应的优化路径。

### 1.1 绝对定位主导型

**特征**：页面中超过半数的元素使用 `position: absolute` 且坐标为硬编码像素值（如 `left: 312px; top: 187px`），元素尺寸也多为固定像素。

**常见问题**：
- 元素重叠/遮挡（字体渲染差异导致文字溢出容器）
- 坐标偏移（转换精度损失导致元素偏离预期位置）
- 无法适应内容量变化（改一处文字可能连锁影响多处）
- z-index 层级混乱

**优化路径**：先执行第二节「绝对定位重构」，再回到第三节做通用排版调优。

### 1.2 流式布局型

**特征**：页面主体使用 flex/grid 布局，元素通过 margin/padding/gap 控制间距，少量或无绝对定位。

**常见问题**：
- 间距不均匀（某些区块挤、某些区块空）
- 字号层次不分明（标题和正文视觉权重接近）
- 布局模式单一（通篇单栏列表）
- 内容偏少、页面空旷

**优化路径**：跳过第二节，直接进入第三节和第四节。

## 二、绝对定位转流式布局

> 仅当第一节诊断为「绝对定位主导型」时执行本节。
>
> 目标是将绝对定位布局重构为 flex/grid 流式布局，使页面具备自适应能力。AI 自行判断如何分组和重构，本节只提供**重构过程中容易踩的坑和必须注意的事项**。

### 2.1 文字处理注意事项

- **固定宽高 + `overflow:hidden` 的文本容器**：原始代码中文本容器通常有固定的 `width`、`height` 和 `overflow:hidden`，这是为了精确还原 PPT 中文本框的尺寸。转为流式布局后，必须去掉固定高度和 `overflow:hidden`，让文字自然撑开，否则字体渲染差异会导致文字被截断
- **spacer 空元素**：可能存在 `display:inline-block; width:Npx` 的空元素，用于模拟 PPT 字符间距。重构时删除这些元素，改用 `letter-spacing` 或 `gap`
- **无效字体名**：`font-family` 中可能出现 `'+mj-ea'`、`'+mn-ea'` 等 `+` 开头的名称，这些是 PPT 内部占位符，浏览器无法解析。从 font-family 列表中移除，保留后续回退字体
- **单字符容器**：数学公式中每个字符可能各占一个独立定位的 div（`display:flex; align-items:center; overflow:visible`，宽度 20–50px），重构时应合并为文本块

### 2.2 数学公式处理注意事项

页面中的分数可能由多个独立定位元素拼合而成：分子文本 div + 分数线 div（`height:1px; border-top:1px solid`）+ 分母文本 div。**重构时必须将它们作为一个整体处理**，不能把分子分母拆散到不同的布局容器中。可以用一个 `position:relative` 的包裹容器将公式组件包起来，内部保持相对定位。

### 2.3 图片处理注意事项

- **`data-img` 延迟加载**：图片可能不直接使用 `src`，而是用 `data-img="N"` 属性配合页面底部的 JS 数组进行延迟赋值。重构时必须保留 `data-img` 属性和对应的 `<script>` 块，否则图片不会加载
- **装饰性形状**：`clip-path:polygon(...)` 实现的三角形、箭头等 CSS 形状，以及半透明色块（`opacity:0.2~0.7`），通常是背景装饰。重构时可保留绝对定位放在背景层（`z-index:-1`），不参与流式布局

### 2.4 动画处理注意事项

这是绝对定位重构中**最容易出错**的部分。

- **`data-anim-id` 属性**：元素上的 `data-anim-id="N"` 是动画控制器的定位标记，页面内的 `<script>` 通过此属性查找元素并控制显示/隐藏。重构时必须保留所有 `data-anim-id` 属性和动画 `<script>` 块
- **动画驱动的内容交替**：PPT 常用"出现/消失"动画来在同一位置交替显示不同内容（如先显示第一步解题过程，点击后第一步消失、第二步出现）。此时多个元素在原始布局中**坐标重叠是有意为之**，它们靠 `visibility:hidden` 初始隐藏，由动画脚本控制依次出现。重构时**不能**把这些重叠元素拆开排列——它们本来就需要占据同一位置。可以用 `position:relative` 的容器包裹、内部元素用 `position:absolute` 叠放
- **判断元素是否参与动画**：如果元素有 `data-anim-id` 属性，说明它受动画控制。如果页面 `<script>` 中的动画序列引用了该 ID 且 `anim_class` 为 `entr`（入场），则该元素初始不可见，后续点击才出现。这类元素的空间**不应计入常规布局**——它们是动画过程中才出现的增量内容

### 2.5 内容总量与溢出

绝对定位页面中，所有内容元素的坐标总高度加起来可能远超 540px，但因为有出场/入场动画控制交替显示，实际任意时刻可见的内容是能装下的。重构为流式布局时，如果把所有元素都平铺展开，极可能溢出。

处理策略：
- **先分析动画序列**，区分"始终可见"和"动画控制显隐"的元素
- 始终可见的元素参与流式布局
- 动画控制显隐的元素用叠放定位（同一位置，靠 JS 切换），或改为逐步展示交互（点击按钮显示下一步）
- 如果去掉动画后内容仍然超出 540px，按第五节的溢出修复优先级处理

## 三、排版规范

### 3.1 字号层级

| 层级 | 推荐字号 | 用途 |
|------|---------|------|
| 大标题 | 28–36px，font-weight: 700 | 页面标题 |
| 小标题 | 20–24px，font-weight: 600 | 分区标题、要点引导 |
| 正文 | 16–18px，font-weight: 400 | 主体内容 |
| 标注 | 12–14px，font-weight: 400 | 注释、来源说明、次要信息 |
| 封面标题 | ≥ 40px | 封面页专用 |

**检查项**：
- 标题与正文的字号差距 ≥ 8px，否则层次不分明
- 同一层级的字号在页面内保持一致
- 标注文字的颜色应比正文浅（降低 opacity 或使用灰色）

### 3.2 间距体系

| 间距类型 | 推荐值 | 说明 |
|---------|--------|------|
| 页面内边距 | 上下 30px、左右 40px | 内容不贴边 |
| 标题与内容间距 | 16–24px | 标题下方留白 |
| 段落间距 | 12–16px | 段落之间 |
| 要点条目间距 | 8–12px | 列表项之间 |
| 模块间距 | 20–32px | 不同内容区块之间 |
| 行高 | 1.5–1.8 | 正文行高 |

**检查项**：
- 页面四周是否有足够的内边距（≥ 20px）
- 模块之间的间距是否大于模块内部的间距（视觉分组原则）
- 是否存在间距为 0 的相邻元素（除非是刻意的紧凑设计）

### 3.3 布局模式选择

根据页面类型和内容量选择合适的布局模式：

| 布局模式 | 适用场景 | 要点 |
|---------|---------|------|
| 单栏满宽 | 讲解页（连续段落）、例题页（题干+解答） | 宽度 ≤ 880px，行长适中 |
| 双栏并排 | 对比、分析（优缺点/异同）、图文对照 | 每栏 ≤ 420px，栏间距 ≥ 24px |
| 左图右文 / 左文右图 | 配图讲解、示意图 + 文字说明 | 图宽 ≤ 50%，文字区保证 ≥ 380px |
| 卡片网格 | 多个并列概念、分类展示 | 2–3 列，卡片内文字 ≤ 80 字/卡 |
| 顶部标题 + 下方内容 | 几乎所有页面的基础结构 | 标题固定在顶部，内容区灵活 |

**检查项**：
- 如果页面只用了单栏列表，考虑是否能换成更丰富的布局（双栏、卡片、图文混排）
- 布局是否与教学意图匹配（对比类内容用双栏，并列概念用卡片网格）

### 3.4 对齐规则

- 页面内的主体内容保持统一的左对齐基线
- 标题可居中或左对齐，同一页面内保持一致
- 封面页标题居中；内容页标题推荐左对齐
- 卡片/要点列表在水平方向上等宽等距
- 避免页面内同时出现左对齐和居中对齐的文字区块（除非标题居中、正文左对齐这种经典组合）

## 四、内容密度评估与充实

### 4.1 密度评估标准

基于前置分析 5.4 的内容清点结果，评估当前页面的信息量：

| 页面类型 | 最低内容要求 | 判定为"偏少"的信号 |
|---------|-------------|-----------------|
| 讲解/精读 | 核心内容 + 要点 ≥ 2 条 | 只有标题 + 一句笼统话；正文 ≤ 100 字且无图表 |
| 例题 | 题干 + 完整解题步骤 | 只有题目没有解答；只有答案没有过程 |
| 练习 | 题干 + 选项/答案 | 只有题目没有选项或反馈 |
| 分析/讨论 | 分析要点 ≥ 2 条 | 只有一个问题，无引导或框架 |
| 小结 | 归纳要点 ≥ 2 条 | 只有"本课小结"四个字 |
| 封面 | 主标题 + 副标题 | 内容充足（封面本身信息量少是正常的） |

### 4.2 内容补充策略

当判定页面内容偏少时，可补充以下类型的内容（按优先级排列）：

1. **展开已有要点**：将过于简短的要点补充为完整的句子或段落
2. **添加教学提示**：如"思考"框、"注意"框、"拓展"框
3. **添加示例或类比**：用具体例子解释抽象概念
4. **添加对比/表格**：结构化呈现信息
5. **添加引用框**：引用原文、定义、公式（须确保引用内容准确）

**禁止**：
- 凭空编造知识点、定义、公式或数据
- 偏离前置分析 5.1 识别出的教学意图
- 添加内容后导致页面溢出——每新增一项内容，须同步评估空间余量

### 4.3 空间预算

新增内容前必须计算剩余空间：

| 参数 | 值 |
|------|-----|
| 画布尺寸 | 960 × 540 px |
| 推荐内边距 | 上下 30px、左右 40px |
| 可用内容区 | 880 × 480 px |
| 页标题占用 | ≈ 56px（含下边距） |
| 标题后剩余高度 | ≈ 424px |
| 每行文字高度（18px 字号） | ≈ 30px（含行距） |

如果剩余高度不足以容纳新增内容，优先缩减已有元素的间距，而非删除内容。

## 五、溢出检测与修复

### 5.1 检测清单

逐项检查以下场景是否存在溢出：

- [ ] 文字内容总高度是否超过 480px 内容区
- [ ] 是否有元素的 `left + width > 960` 或 `top + height > 540`
- [ ] 交互展开后（如显示答案、展开解析）的终态是否溢出
- [ ] 动画终态是否有元素移出画布
- [ ] 图片是否有 `width` 或 `height` 超出画布的情况

### 5.2 修复优先级

发现溢出时，按以下优先级处理（优先做影响小的调整）：

1. **缩减间距**：减小 padding、margin、gap（不低于第三节的最小推荐值）
2. **缩小字号**：正文降至 16px，标注降至 12px（不低于 12px）
3. **缩小图片**：等比缩小，保留宽高比
4. **切换布局**：将纵向排列改为横向并排，或改为更紧凑的布局模式
5. **精简内容**：合并重复要点、缩短过长的说明文字（须保留核心信息）
6. **分层展示**：将部分内容改为点击展开/标签页切换（须实现 saveState/restoreState）

**禁止**：
- 将 overflow 设为 hidden 来"隐藏"溢出内容
- 删除教学核心内容来腾出空间
